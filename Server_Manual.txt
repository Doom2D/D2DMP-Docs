"Инструкция по эксплоатации" сервера Doom 2D Multiplayer 0.6 (Server.exe)
Написал Чёрный Думер.
=*=*=*=*=*=*=*=*=*=
СОДЕРЖАНИЕ:
1) Описание окна сервера
2) Описание консольных команд сервера
3) Описание конфигурационных файлов сервера
4) Описание параметров конфигурации сервера
5) Чего НЕ РЕКОМЕНДУЕТСЯ делать
=*=*=*=*=*=*=*=*=*=
1) ОПИСАНИЕ ОКНА СЕРВЕРА
Окно сервера представляет собой консоль с серой полоской в самом низу, на которой отображаются некоторые данные о сервере - его название, имя файла запущенной на нём карты, число игроков на сервере, его внешний IP-адрес, оставшееся время раунда (в секундах) и счётчик FPS (кадров в секунду).
* * * * * * * * * *
2) ОПИСАНИЕ КОНСОЛЬНЫХ КОМАНД СЕРВЕРА
clist - вывести список клиентов (вместе с ботами)
echo STR - вывести строку STR в консоль
exit/quit - выключить сервер
help - вывести список консольных команд с описаниями (неполный)
kick X - отключить клиента из слота X
ban X - занести IP клиента из слота X в банлист
ban_reload - перезагрузить банлист
bot_add - добавить бота
say STR - написать строку STR в чат
bot_tickrate 1-10 - как часто боты будут передавать свои координаты и производиться синхронизация незначительных событий вроде эффекта дыма от Jetpack'а
bot_randrate 1-10 - вкл/выкл преобладание функции random() в поведении ботов
bot_cowardly 1/0 - будет ли убегать бот при маленьких HP и прятать ракетницу и BFG при близком контакте
cl_timeout 0-3600 - установить время до отключения AFK (или вылетевших) игроков
sv_map ИМЯ_ФАЙЛА_С_КАРТОЙ - сменить карту
sv_maxplayers 0-16 - установить максимальное количество слотов
sv_use_pwd 0/1 - требовать ли с игроков пароль
sv_password ПАРОЛЬ - установить пароль, требуемый с игроков
sv_lan 1/0 - запрашивать ли внешний IP-адрес с whatismyip.org
sv_rcon 1/0 - позволять ли клиентам использовать rcon
sv_rcon_pwd ПАРОЛЬ - установить пароль, который необходимо требовать с клиентов для использования rcon
sv_clalert 1/0 - вкл/выкл проигрывание звука при подключении клиента	
mp_fraglimit 0-200 - установить фраглимит
mp_respawn 0-60 - установить время между гибелью и респауном игрока (в секундах)
mp_itemrespawn 0-360 - установить время между взятием и респауном предметов (в секундах)
mp_items 0/1 - вкл/выкл спаун предметов
mp_powerups 0/1 - вкл/выкл спаун пауер-апов
mp_knockback 0/1 - будут ли отбрасывать игроков взрывы
mp_selfdamage 0/1 - вкл/выкл урон от своих взрывов и кислоты
mp_flymode 0/1 - вкл/выкл режим полета для всех игроков (не сохраняется в файл конфигурации)
mp_godmode 0/1 - вкл/выкл режим бессмертия для всех игроков (не сохраняется в файл конфигурации)
mp_oldaim 0/1 - вкл/выкл углы подъёма оружия вверх/вниз как в Doom 2D
* * * * * * * * * *
3) ОПИСАНИЕ КОНФИГУРАЦИОННЫХ ФАЙЛОВ СЕРВЕРА
Все конфигурационные файлы сервера находятся в папке data/cfg.
server.cfg - основной конфигурационный файл сервера, определяющий его параметры
autoexec.cfg - список консольных команд, выполняемых сразу же после загрузки карты (туда можно запихивать, например, bot_add)
bot_chatter.txt - реплики, которые "пишут" боты в чат
bot_names.txt - имена ботов
ip_bans.txt - банлист (список забаненных IP-адресов)
map_list.txt - карты, которые сервер загружает при каждом новом раунде
* * * * * * * * * *
4) ОПИСАНИЕ ПАРАМЕТРОВ КОНФИГУРАЦИИ СЕРВЕРА
sv_map ИМЯ_ФАЙЛА_С_КАРТОЙ - карта, запускаемая на сервере в первом раунде
sv_name NAME - установить имя сервера на NAME
sv_welcome STR - установить приветственное сообщение сервера на STR
sv_maxplayers 0-16 - установить максимальное количество слотов
sv_clalert 0/1 - вкл/выкл проигрывание звука при подключении клиента
sv_use_pwd 0/1 - требовать ли с игроков пароль
sv_password ПАРОЛЬ - установить пароль, требуемый с игроков
sv_lan 0/1 - запрашивать ли внешний IP-адрес с whatismyip.org (влияет на серверлист и проверку портов на доступность)
sv_portcheck 0/1 - вкл/выкл проверку портов на доступность
sv_rcon 0/1 - позволять ли клиентам использовать rcon
sv_rcon_pwd ПАРОЛЬ - установить пароль, требуемый с клиентов для использования rcon
sv_ipbans 0/1 - вкл/выкл загрузку банлиста из ipbans.txt
sv_cycle_maps 0/1 - вкл/выкл загрузку списка карт для каждого нового раунда из map_list.txt
sv_cheats 0/1 - вкл/выкл читы
sv_slist_time 0-80 - задержка перед следующей синхронизацией с серверлистом (оптимальное значение 60) (в секундах)
sv_slist ADR - адрес хоста, на котором находится серверлист
cl_timeout 0-3600 - установить время до отключения AFK (или вылетевших) игроков
mp_fraglimit 0-200 - установить фраглимит
mp_timelimit 0-6000 - установить время одного раунда (в секундах)
mp_respawn 0-60 - установить время между гибелью и респауном игрока (в секундах)
mp_itemrespawn 0-360 - установить время между взятием и респауном предметов (в секундах)
mp_items 0/1 - вкл/выкл спаун предметов
mp_powerups 0/1 - вкл/выкл спаун пауер-апов
mp_knockback 0/1 - будут ли отбрасывать игроков взрывы
mp_selfdamage 0/1 - вкл/выкл урон от своих взрывов и кислоты
mp_oldaim 0/1 - вкл/выкл углы подъёма оружия вверх/вниз как в Doom 2D
mp_itemdrop 0/1 - вкл/выкл вообще выброс оружия игроком после его гибели
mp_dropall 0/1 - вкл/выкл выброс всего оружия из игрока после его смерти
mp_weaponstay 0/1 - остаётся ли оружие на своём месте после подбора (также отключает выброс оружия игроками после гибели, если он включён)
mp_danmaku 0/1 - вкл/выкл данмаку-режим
bot_names 0/1 - вкл/выкл загрузку имён ботов из bot_names.txt
bot_chatter 0/1 - вкл/выкл загрузку чат-фразы ботов из bot_chatter.txt
bot_tickrate 1-10 - как часто боты будут передавать свои координаты и производиться синхронизация незначительных событий вроде эффекта дыма от Jetpack'а
bot_randrate 1-10 - вкл/выкл преобладание функции random() в поведении ботов
bot_cowardly 0/1 - будет ли убегать бот при маленьких HP и прятать ракетницу и BFG при близком контакте
* * * * * * * * * *
5) ЧЕГО НЕ РЕКОМЕНДУЕТСЯ ДЕЛАТЬ
Список тех вещей, которые очень и очень не рекомендуется делать:
 - ставить больше 5-7 ботов (особенно на больших картах);
 - запускать множество других программ, если работает сервер (особенно при наличии большого числа подключённых клиентов);
 - ставить нулевой или отрицательный фраглимит/лимит времени раунда;
 - записывать в bot_chatter.txt слишком много реплик;
 - записывать в bot_names.txt больше 16 имен (DEFAULT не в счёт), всё равно они не загрузятся.
