1) ОПИСАНИЕ ОКНА СЕРВЕРА
Окно сервера представляет собой консоль с серой полоской в самом низу, на которой отображаются некоторые данные о сервере - его название, имя файла запущенной на нём карты, число игроков на сервере, его внешний IP-адрес, оставшееся время раунда (в секундах) и счётчик FPS (кадров в секунду).
* * * * * * * * * *
2) ОПИСАНИЕ КОНСОЛЬНЫХ КОМАНД СЕРВЕРА
clist - вывести список клиентов (вместе с ботами)
echo STR - вывести строку STR в консоль
exit/quit - выключить сервер
help - вывести список консольных команд с описаниями (неполный)
kick X - отключить клиента из слота X
ban X - занести IP клиента из слота X в банлист
ban_reload - перезагрузить банлист
bot_add N S C - добавить бота с псевденимом N, скином S и цветом брони C
	Цвет определяется его TrueColor-индексом (диапазон: 0-16777215)
	Если вместо одного из значений поставить символ [*] ("звёздочка"), то оно задастся случайно. То же самое произойдёт при неправильном значении.
	Если же вообще не указывать ни одно из значений, то они все зададутся случайно.
		Инструкция по конвертированию RGB-значений в TrueColor-индекс:
		1. Переводим R, G и B значения в двухзначный шестнадцатеричный вид;
		2. Записываем полученные значения в таком порядке: B, G, R;
		3. Конвертируем полученную строку обратно в десятичный вид.
		4. Готово.
		Пример: у нас есть R[3], G[5], B[7]
		Переводим их в двузначные шестнадцатеричные значения, получаем R[03], G[05], B[07]
		Записываем их в обратном порядке (B>G>R), получаем [070503]
		Переводим обратно в десятичное, получаем [460035]
bot_kick - удалить всех ботов с сервера
say STR - написать строку STR в чат
resync - произвести синхронизацию с серверлистом
cl_setval ID ARG VAL - изменить значение аргумента ARG у игрока из слота ID на значение VAL
	Аргументы, значения которых можно изменить:
	hp - здоровье
	ap - броня
	frag - количество фрагов
	jet - наличие Jetpack'а (VAL: 0 - нету, 1 - есть)
	inv - наличие неуязвимости (VAL: 0 - нету, 1 - есть)
	ber - наличие берсерка (VAL: 0 - нету, 1 - есть)
	bpk - наличие рюкзака (VAL: 0 - нету, 1 - есть)
	wX - наличие оружия (вместо X - номер оружия) (VAL: 0 - нету, 1 - есть)
	aX - количество патронов (вместо X - номер типа патронов) (VAL: 0 - нету, 1 - есть)
	ВНИМАНИЕ: данные таким образом игроку Jetpack или неуязвимость будут работать, пока игрок не погибнет!
%параметр_конфигурации_сервера% - показать значение введённого параметра
%параметр_конфигурации_сервера% %значение% - присвоить введённому параметру введённое значение (не для всех параметров)
* * * * * * * * * *
3) ОПИСАНИЕ КОНФИГУРАЦИОННЫХ ФАЙЛОВ СЕРВЕРА
Все конфигурационные файлы сервера находятся в папке data/cfg.
server.cfg - основной конфигурационный файл сервера, определяющий его параметры
autoexec.cfg - список консольных команд, выполняемых сразу же после загрузки карты (туда можно запихивать, например, bot_add)
bot_chatter.txt - реплики, которые "пишут" боты в чат
bot_names.txt - имена ботов
bot_skins.txt - список скинов, которые будут использовать боты
ip_bans.txt - банлист (список забаненных IP-адресов)
map_list.txt - карты, которые сервер загружает при каждом новом раунде
* * * * * * * * * *
4) ОПИСАНИЕ ПАРАМЕТРОВ КОНФИГУРАЦИИ СЕРВЕРА
sv_map NAME - имя файла карты (без расширения), запускаемой на сервере (NAME - изменяемый аргумент)
sv_name STR - название сервера (STR - изменяемый аргумент)
sv_welcome STR - приветственное сообщение при входе на сервер (STR - изменяемый аргумент)
sv_maxplayers 0-16 - максимум игроков на сервере
sv_use_pwd 0-1 - вкл/выкл использование пароля
sv_password PWD - пароль сервера (PWD - изменяемый аргумент)
sv_lan 0-1 - вкл/выкл запрос внешнего IP-адреса с whatismyip.org, видимость сервера в серверлисте и проверку портов на доступность (для игр по локальной сети)
sv_portcheck 0-1 - вкл/выкл проверку портов на доступность
sv_rcon 0-1 - вкл/выкл использование RCON
sv_rcon_pwd PWD - RCON-пароль сервера (PWD - изменяемый аргумент)
sv_ipbans 0-1 - вкл/выкл загрузку банлиста из ipbans.txt
sv_cycle_maps 0-2 - вкл/выкл загрузку списка карт для каждого нового раунда из map_list.txt
	Возможные параметры:
	0 - список карт выключен, каждый новый раунд начинается на одной и той же карте, предварительно заданной в sv_map
	1 - список карт включён, карты чередуются строго по порядку
	2 - список карт включён, карты выбираются случайно
sv_cheats 0-1 - вкл/выкл чит-коды
sv_clalert 0-1 - вкл/выкл сигнал при подключении клиента
sv_slist_time 5-60 - задержка перед следующей синхронизацией с серверлистом (в секундах)
sv_slist ADR - адрес хоста, на котором находится серверлист (например d2dmp.ru) (ADR - изменяемый аргумент)
sv_slist_path ADR - путь к PHP-скриптам серверлиста (например /slist/) (ADR - изменяемый аргумент)
cl_rc_time 0-60 - задержка до переподключения игроков после конца раунда (в секундах)
cl_timeout 0-3600 - задержка до отключения AFK (или вылетевших) игроков (в секундах)
mp_fraglimit 0-250 - фраглимит
mp_timelimit 0-9000 - время одного раунда (в секундах)
mp_respawn 0-60 - задержка до респауна убитого игрока (в секундах)
mp_respawn_inv 0-10 - время действия неуязвимости у игрока после респауна (в секундах)
mp_itemrespawn 0-300 - задержка до респауна взятого предмета (в секундах)
mp_drop_clear 0-300 - задержка до удаления выброшенного убитым игроком оружия (в секундах)
mp_items 0-1 - вкл/выкл спаун предметов
mp_powerups 0-1 - вкл/выкл спаун пауерапов
mp_knockback 0-1 - вкл/выкл отбрасывание игроков от взрывов
mp_selfdamage 0-1 - вкл/выкл урон от взрывов своих ракет и кислоты
mp_gamemode 0-3 - режим игры (не используется)
mp_oldaim 0-1 - вкл/выкл углы подъёма оружия вверх/вниз как в Doom 2D
mp_itemdrop 0-1 - вкл/выкл вообще выброс оружия игроком после его гибели
mp_dropall 0-1 - вкл/выкл выброс всего оружия из игрока после его смерти
mp_weaponstay 0-1 - вкл/выкл режим, когда оружие остаётся на своём месте после подбора (также отключает вообще выброс оружия игроками после гибели, если он включён)
mp_penalty 0-1 - вкл/выкл вычет фрагов за самоубийства
bot_names 0-1 - вкл/выкл загрузку имён ботов из bot_names.txt
bot_chatter 0-1 - вкл/выкл загрузку чат-реплик ботов из bot_chatter.txt
bot_tickrate 0-10 - как часто боты будут передавать свои координаты и будет производиться синхронизация незначительных событий вроде эффекта дыма от Jetpack'а
bot_randrate 0-8 - сила преобладания функции random() в поведении ботов
bot_cowardly 0-1 - вкл/выкл режим, когда бот будет убегать при маленьких HP и менять ракетницу на другое оружие при близком контакте
bot_dummy 0-1 - вкл/выкл искусственный интеллект (ИИ) у ботов

5) ЧЕГО НЕ РЕКОМЕНДУЕТСЯ ДЕЛАТЬ
Список тех вещей, которые очень и очень не рекомендуется делать:
 - ставить больше 5-7 ботов (особенно на больших картах);
 - запускать множество других программ, если работает сервер (особенно при наличии большого числа подключённых клиентов);
