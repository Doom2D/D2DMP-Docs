1. Что это?
Пре-релизная версия Doom 2D Multiplayer v0.6, очевидно же. 
Отсутствуют некоторые важные (и не очень) вещи вроде серверлиста и меню отключения от текущей игры.

2. Как играть
Запускаем Server.exe. Это сервер. Если машина слабая, лучше вообще не играть. :3
Запускаем Doom2DMP.exe. Настраиваем по своему вкусу. Про настройки сервера см. в соотв. секции. Жмем "Начать игру", вводим 127.0.0.1 в поле IP. И поехали.

3. 1. Чего лучше не делать:
- ставить больше пяти-семи ботов. Особенно на больших картах;
- стрелять много из пулеметов/шотганов на больших картах;
- запускать в фоне хуеву тучу программ, если работает и клиент, и сервер;
- ставить нулевой или отрицательный таймлимит/фраглимит;
- играть с пингом выше 100;
- делать скины с как-нибудь исковерканными файлами анимаций;
- удалять текстуры из папки \gui;
- делать файл bot_chatter.txt слишком большим;
- вводить в bot_names.txt больше 16 имен (кроме DEFAULT), все равно они не загрузятся.

3. 2. Известные баги:
Спросите Черного и ar888.

4. Консольные команды и прочие фишки:
Клиент:
exit/quit - выход
name NAME - сменить имя
color R G B - сменить цвет
skin SKIN_FOLDER_NAME - сменить скин
disconnect - вывалиться из текущей игры
help - вывод некоторых команд
bind KEY COMMAND - забиндить что-то куда-то
bind_info - информация о биндах
echo ТЕКСТ - вывести текст в консоль
r_gfx 0/1 - убирает эффекты вроде искр, крови, трупов, вспышек, инверсии цветов и так далее
r_massacre 0-3 - уровень насилия (0 - нет, 1 -только кровь, 2 - кровь и трупы, 3 - кровь, трупы и куски)
r_names 0/1 - рисовать имена игроков над головой
r_window 0/1 - оконный режим
s_preload 0/1 - грузить сразу все звуки при старте клиента
s_vol_sound 0-100 - громкость звука
s_vol_music 0-100 - громкость музыки
exec CFG - открывает конфиг
sv_password PASS - пароль, который посылать серверу
sv_rcon_pwd PASS - пароль rcon
rcon CMD - выполнить команду CMD на сервере (если пароль rcon совпадает с серверным)
cls - очистить консоль

Сервер:
exit/quit - выход
echo ТЕКСТ - вывести текст в консоль
fps - отображает FPS
clist - выводит список клиентов
kick ID - кикнуть клиента ID
ban ID - занести IP клиента ID в банлист
ban_reload - перезагрузить банлист
sv_map КАРТА - сменить карту
sv_name ИМЯ - сменить имя сервера
sv_welcome - сменить приветственное сообщение сервера
sv_maxplayers 0-16 - количество слотов
sv_use_pwd 0/1 - требовать ли с игроков пароль
sv_password ПАРОЛЬ - какой пароль требовать
sv_lan 1/0 - запрашивать ли IP с whatismyip.org
sv_rcon 1/0 - позволять ли клиентам использовать rcon
sv_rcon_pwd ПАРОЛЬ_РКОН - пароль, который требовать с них для использования rcon
sv_ipbans 1/0 - грузить ли баненные IP из файла ipbans.txt
sv_clalert 1/0 - проигрывать звук при подключении клиента
sv_cycle_maps 1/0 - менять ли карты автоматом по истечению времени/фраглимита (map_list.txt)
cl_timeout 0-3600 - через сколько секунд кикать AFK (или вылетевших) игроков	
mp_fraglimit 0-200 - фраглимит
mp_timelimit 0-6000 - таймлимит (сек)
mp_respawn 0-60 - время респауна игроков (сек)
mp_itemrespawn 0-360 - время респауна итемов (сек)
mp_items 0/1 - спаунить ли итемы
mp_powerups 0/1 - спаунить ли поверапы
mp_knockback 0/1 - отбрасывает ли игрока взрывами
mp_selfdamage 0/1 - получать ли урон от своих взрывов и кислоты
mp_flymode 0/1 - режим полета для всех игроков
mp_godmode 0/1 - режим бессмертия для всех игроков
mp_oldaim 0/1 - прицеливание как в D2DF
bot_add - добавить бота
bot_names 0/1 - грузить имена ботов из bot_names.txt
bot_chatter 0/1 - грузить и использовать чат-фразы ботов из bot_chatter.txt
bot_tickrate 1-10 - как часто бот передает свои координаты (а так же как часто производится синхр-я незначительных событии вроде дымка из жопы)
bot_randrate 1-10 - преобладание функции random() в поведении ботов
bot_cowardly 1/0 - убегает ли бот при маленьких HP, прячет ли при близком контакте ракетницу и BFG

Редактор:
1-8 - показать слои:
1 - фон
2 - ступени
3 - стены
4 - точки DM
5 - итемы
6 - лифты
7 - жидкости
8 - передний план
Зажать Q - объекты не будут удалять друг друга при наложении (очень не рекомендую использовать с шифтом. Объекты будут создаваться друг на друге 60 раз в секунду, а это очень неприятно) 
Зажать Е - режим предпросмотра


5. Отличия от build 111:
- при выключенной сетке в редакторе клавишами WASD можно попиксельно двигать тайлы;
- размер игрока теперь точно равен 35х52, и в редакторе тоже;
- сервер проигрывает звук при подключении клиента (sv_clalert);
- после загрузки карты сервер выполняет команды из autoexec.cfg;
- если у игрока при смерти в руках пистолет/кулак, перед смертью он меняет оружие на более сильное (если есть), дабы выкинуть его, если надо;
- у панелей без текстуры рисуются разноцветные контуры;
- контуры бестекстурных панелей исчезают в режиме предосмотра;
- клиент отваливается от сервера, если не получает никаких сообщений в течение 30 секунд;
- клиент отваливается от сервера при получении мусорных пакетов (например, если сервер закрыть по CTRL-ALT-DEL);
- в Credits нижний блок текста поднят на 4 пикселя;
- боты больше не говорят "0" вместо фразы случайным образом;
- клиент удаляет из памяти старые текстуры, фоны и музыку, если зайти на сервер повторно/сменится карта;
- на Inter рисуется название и описание карты;
- сервер не будет грузить карту без точек ДМ;
- поправлены несколько багов в картах.
